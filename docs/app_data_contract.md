# App Data Contract

This document defines the first app-facing data files generated by the player-season export pipeline. The app should read these processed files instead of raw NBA Stats endpoint caches.

## Files

- `app_player_profiles_<season>.csv`: one row per player-season for search, profile pages, and comparison cards.
- `app_similarity_edges_<season>.csv`: nearest-neighbor edges for similarity lists and graph views.
- `app_feature_metadata_<season>.csv`: feature labels, groups, descriptions, and display hints for controls and tooltips.

## Player Profiles

The profile table combines identity, top similarity outputs, selected scoring-style features, source availability flags, and a short style summary.

Important columns:

- `season`
- `player_id`
- `player_name`
- `team_abbreviation`
- `position`
- `primary_position`
- `games_played`
- `minutes_total`
- `minutes_per_game`
- `points_per_36`
- `usage_rate`
- `true_shooting_pct`
- `top_neighbor`
- `top_neighbor_similarity`
- `top_archetype`
- `top_archetype_similarity`
- `top_position_reference`
- `top_position_similarity`
- `style_summary`
- `has_synergy_data`
- `has_shot_location_data`
- `has_tracking_shot_data`
- `has_touch_tracking_data`

Selected scoring-style columns are included for profile cards, scatter plots, and filters:

- isolation and pick-and-roll frequency/efficiency
- spot-up, cut, handoff, post-up, off-screen, and transition frequency/efficiency
- rim, mid-range, corner-three, and above-break-three attempt rates
- catch-and-shoot and pull-up frequency/efficiency
- drive, touch, passing, and average seconds-per-touch features

## Similarity Edges

The similarity edge table is long-form: one row per player-neighbor pair.

Important columns:

- `season`
- `player_id`
- `player_name`
- `team_abbreviation`
- `neighbor_rank`
- `neighbor_player_id`
- `neighbor_player_name`
- `neighbor_team_abbreviation`
- `cosine_similarity`

The default export writes five neighbors per player when `--top-n 5` is used.

## Feature Metadata

The metadata table supports app controls and explanations.

Important columns:

- `feature`: source column name.
- `label`: display label.
- `group`: UI grouping such as `scoring`, `play_type`, `shot_location`, or `tracking_shot`.
- `description`: short display description.
- `higher_is`: display hint such as `more`, `better`, `contextual`, or `coverage`.
- `is_fingerprint_dimension`: whether the feature is part of similarity scoring.

Source availability flags are intentionally marked as non-fingerprint dimensions.
